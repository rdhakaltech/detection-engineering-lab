# Detection Engineering Lab

## Project Overview

`detection-engineering-lab` is a public portfolio project that demonstrates practical detection engineering concepts using simulated security telemetry. The lab includes example SIEM detection logic, fake Windows and authentication logs, MITRE ATT&CK mappings, triage guidance, and a basic automation script for generating sample process creation events.

All data in this repository is simulated. No real company, customer, employee, endpoint, or production security data is included.

## Purpose

This project is designed to showcase a structured detection engineering workflow suitable for a security engineering portfolio. It focuses on how detections are documented, mapped, tested conceptually, tuned for false positives, and supported with analyst triage guidance.

## Key Focus Areas

- SIEM detection logic using generic SQL-style queries
- Windows process creation analysis using Event ID 4688-style telemetry
- Suspicious command-line pattern detection
- PowerShell abuse detection concepts
- False positive review and tuning
- MITRE ATT&CK technique mapping
- Threat hunting and investigation workflow
- Basic automation for generating simulated lab data

## Folder Explanation

### `detections/`

Contains documented detection logic for common attacker behaviors. Each detection includes the goal, data source, generic SIEM query, false positives, tuning recommendations, MITRE ATT&CK mapping, triage steps, and severity.

### `sample_logs/`

Contains realistic but fake JSON logs that can be used to understand the expected event structure and test basic parsing workflows.

### `triage_guides/`

Contains analyst-facing investigation guidance for reviewing alerts generated by the detections.

### `scripts/`

Contains simple automation scripts for generating simulated log data.

### `mappings/`

Contains MITRE ATT&CK mappings that connect each detection to relevant adversary techniques.

## Featured Detections

### Encoded PowerShell Execution

Detects suspicious PowerShell command lines using encoded commands, hidden windows, disabled profiles, and inline execution patterns such as `iex`.

Relevant techniques include:

- T1059.001 - PowerShell
- T1027 - Obfuscated Files or Information
- T1140 - Deobfuscate/Decode Files or Information

### Suspicious Command-Line Binary Usage

Detects potentially abusive use of native Windows binaries commonly used for download, proxy execution, or defense evasion behavior, including:

- `certutil`
- `bitsadmin`
- `mshta`
- `rundll32`

Relevant techniques include:

- T1105 - Ingress Tool Transfer
- T1218.005 - Mshta
- T1218.011 - Rundll32
- T1197 - BITS Jobs

## Getting Started

Generate fresh fake Windows Event ID 4688-style logs:

```bash
python3 scripts/log_generator.py
```

The script writes simulated output to:

```text
sample_logs/generated_windows_event_4688.json
```

## Notes

- Queries are intentionally generic and may need syntax adjustments for a specific SIEM.
- Log fields are simplified for readability and portability.
- Detections are designed for educational and portfolio purposes.
- This repository does not include proprietary rules, private detection logic, or real incident data.
